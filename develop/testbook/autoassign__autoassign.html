1. Autoassign by Publisher Account

Test context
  • role: publisher, admin
Setup:

Run the autoassign testdrive at /testdrive/autoassign if you have not already done so

ID Action Expected Result  
R.1.1 Using the details under `map_by_account` in the testdrive setup, log in using the `publisher_username` and `publisher_password`

Application Link: http://testdoaj.cottagelabs.com//account/login 		   
R.1.2 Go to the Publisher's list of Update Requests    
R.1.2.1   There is one update request in the list, and you have an Edit button available
R.1.3 Click on the Edit button for the update request    
R.1.4 Make a minor change to the form data (e.g. change a URL or a title), then go to the final page of the form, select the confirmation checkbox, and click Submit.    
R.1.4.1   Your update request is saved and you return to the Publisher's list of Update Requests
R.1.5 Log out of the Publisher account    
R.1.6 Using the details under `map_by_account` in the testdrive setup, log in using the `maned_username` and `maned_password`    
R.1.6.1   You are taken to the Managing Editor's dashboard
R.1.6.2   There is a card on the dashboard for an Update Request, with the title of the journal you just edited
R.1.6.3   The editor group for the card is identified as "Account Mapping" (along with some random characters)
R.1.7 Click on the Update Request card    
R.1.7.1   You are taken to the Update Request form for the journal you just edited
R.1.7.2   The Group field is set to "Account Mapping" (along with some random characters)
R.1.7.3   There is no Individual editor set
R.1.7.4   There is a note with a recent timestamp indicating that this update request was autoassigned, and the reason for the assignment
R.1.8 Click "Unlock & Close" then close the tab    
R.1.9 Log out of the Managing Editor account

4. Autoassign import failure alert

Test context
  • role: admin
Setup:

Run the autoassign testdrive at /testdrive/autoassign if you have not already done so

ID Action Expected Result  
R.4.1 Using the details under `map_by_account` in the testdrive setup, log in using the `admin_username` and `admin_password`    
R.4.2 Go to the Admin Dashboard

Application Link: http://testdoaj.cottagelabs.com//dashboard 		   
R.4.2.1   On the left navigation, there is a menu item with an alert triangle icon
R.4.3 Click on the menu item with the alert triangle icon

Application Link: http://testdoaj.cottagelabs.com//admin/alerts 		   
R.4.3.1   You are taken to the Admin Alerts page
R.4.3.2   The "new" alert filter is already selected
R.4.3.3   There is an alert from "Autoassign test" with a recent date
R.4.4 Click "Set In Progress"    
R.4.4.1   A pop-up tells you the operation was successful and to reload the page
R.4.5 Reload the page    
R.4.5.1   The alert has disappeared
R.4.6 Clear the "new" filter using the X next to the filter name    
R.4.6.1   All alerts are now visible, including the one you just set to "In Progress" (there may only be this one alert in the test system)
R.4.6.2   The alert has only one option "Mark Closed"
R.4.6.3   The alert has an audit trail showing that it was set to "In Progress" by the user
R.4.7 Click "Mark Closed"    
R.4.7.1   A pop-up tells you the operation was successful and to reload the page
R.4.8 Reload the page    
R.4.8.1   The alert is still visible, but now has a status of "Closed"
R.4.8.2   There are no buttons available on the alert.
R.4.8.3   The alert has an audit trail showing that it was marked closed by the user
