ID Action Expected Result

3. Select a pre-determined reject reason only

Test context
  • role: admin
ID Action Expected Result  
BT.3.1 Go to the "Applications/Update Requests" page in the Administration area    
BT.3.2 Select an application that is not either "Accepted" or "Rejected" (e.g. by selecting "Pending" from the "Application Status" facet on the left), and click Review application    
BT.3.2.1   The applicaiton review page is shown
BT.3.2.2   The "Quick Reject" feature is shown above the application form
BT.3.3 Select a reason for rejection from the pull-down menu. Select any reason except "Other". Do not enter any text in the additional information box    
BT.3.4 click "Quick Reject"    
BT.3.4.1   A flash message containing the text "This application was rejected with the reason 'X'" is displayed, where X is the reason you selected for rejection.
BT.3.4.2   A flash message containing the text "Sent email to 'X' to tell them that their journal application was rejected" is displayed, where X is the journal contact
BT.3.4.3   The applicaiton review page is shown
BT.3.4.4   The application is in status "Rejected"
BT.3.4.5   There is no "Quick Reject" button shown on the page
BT.3.4.6   There is a note attached to the application at the current date/time which contains the text "[User name:] This application was rejected with the reason 'X'" is displayed, where X is the reason you selected for rejection.
BT.3.5 Click the "Unlock and Close" button
