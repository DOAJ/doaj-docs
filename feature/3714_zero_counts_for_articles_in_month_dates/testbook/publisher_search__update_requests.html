1. Test Publisher Update Request Search Interactions

Test context
  • role: publisher
ID Action Expected Result  
BP.1.1 Go to the DOAJ admin applications search at /publisher/progress    
BP.1.1.1   All available records are found, and the first page of results is shown
BP.1.1.2   Results are sorted by last updated date, oldest first
BP.1.1.3   On the left facet "Application Status" is open
BP.1.2 Open one of the facets of your choice    
BP.1.3 Select one of the values for that facet    
BP.1.3.1   All the results for the selected constraints are found, and the first page is shown
BP.1.3.2   The filter applied is shown in the facets on the left and above the search results
BP.1.4 Go through each sort option in the "sort by" pull-down    
BP.1.5 In each case, click the "sort by" button to reverse the direction of sort    
BP.1.5.1   The current search results are ordered correctly according to the selected sort field and the selected sort direction
BP.1.6 Enter a search term in the search box    
BP.1.6.1   All the results for the selected constraints are found, and the first page is shown
BP.1.6.2   Your search text ramains in the search box
BP.1.7 Go through each field constraint under the "search all" pull-down. You may change your search text if no results are found, to demonstrate that each field works correctly    
BP.1.7.1   If there are search results, the first page is displayed
BP.1.7.2   If there are no search results, suitable "no results match" text is displayed
BP.1.8 Change the page size, using the page-size pull-down    
BP.1.8.1   The size of the page of results displays increases appropriately
BP.1.9 Click the "Next" button to go to the next page of results    
BP.1.9.1   The next page of results is displayed
BP.1.9.2   The "First" and "Prev" buttons are activated
BP.1.10 Click "Next" again    
BP.1.11 Click "Prev"    
BP.1.11.1   You are navigated one page forward and then one page back in the results
BP.1.12 Click "First"    
BP.1.12.1   You are taken to the first page of the results
BP.1.13 Click the Red X button to clear all filters    
BP.1.13.1   The interface returns to the initial state as per 1.1.1

2. Test Admin Application Search Display

Test context
  • role: admin
ID Action Expected Result  
BP.2.1 Go to the DOAJ admin application search at /admin/applications    
BP.2.2 Open the "Application Status" facet and select "Accepted"    
BP.2.2.1   Every result includes the text "This update request has been accepted, and your journal in DOAJ updated (view request)." on the bottom right
BP.2.3 Click on "View request" for one of the results    
BP.2.3.1   You are taken to the edit form for the application but it is read only (there is a warning displayed at the top of the page "Please note this form is read-only, you cannot make changes here."
BP.2.4 Click "Back"    
BP.2.5 Clear the "Accepted" filter and select "Rejected"    
BP.2.5.1   Every result includes the text "This update request has been rejected, and your journal in DOAJ updated (view request)." on the bottom right
BP.2.6 Click "Back"    
BP.2.7 Clear the "Accepted" filter and select any status other than "Accepted" or "Rejected"    
BP.2.7.1   Every result includes the text "This update request is currently being reviewed by an Editor (view request)." on the bottom right
BP.2.8 Click on "View request" for one of the results    
BP.2.8.1   You are taken to the edit form for the application but it is read only (there is a warning displayed at the top of the page "Please note this form is read-only, you cannot make changes here."
