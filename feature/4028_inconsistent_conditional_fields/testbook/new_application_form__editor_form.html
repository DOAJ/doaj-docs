  • role: editor
ID Action Expected Result

BH.1.2.1   The editor view of the application form is displayed
BH.1.2.2   "Unlock and Close" is available on the top right

BH.1.2.4   A "SAVE" button is available

BH.1.2.6   The main application form questions are available

2. Note features for Editor

Test context
  • role: editor
ID Action Expected Result  
BH.2.1 Prepare two editor accounts (editor1 and editor2), they both can access to the same application (e.g. /editor/application/1234)    
BH.2.2 Login as `editor1`    
BH.2.3 Go to Application page at `/editor/application/1234`    
BH.2.4 Click "ADD" button in "Notes" area    
BH.2.4.1   New note UI is displayed for you to fill in
BH.2.5 Fill in the note    
BH.2.6 Click "Save" button    
BH.2.6.1   The note is saved and displayed in the list of notes
BH.2.6.2   you username and user id will be showed on that note
BH.2.6.3   the date and time you created the note will be shown
BH.2.7 Click into the note field and edit the note    
BH.2.7.1   The note is editable
BH.2.8 Click "Remove" underneath the note    
BH.2.8.1   The note is deleted
BH.2.9 Create a new note as per the above steps    
BH.2.10 Switch to other associate editor account `editor2`    
BH.2.11 Go to Application page at `/editor/application/1234`    
BH.2.11.1   You can see the note that you have created before with `editor1`

BH.2.13 Attempt to click the "Remove" button

BH.2.14 Click "copy" button next to one of the fields (eg. Title)

BH.2.15 Attempt to paste the value (use separate editor)
