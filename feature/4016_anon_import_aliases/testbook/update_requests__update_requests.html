1. Submitting an update

Test context
  • role: Publisher
ID Action Expected Result  
BU.1.1 Go to the My journals tab on the Publisher dashboard    
BU.1.1.1   You see a list of journals, each with a button "Update"
BU.1.2 Click the "Update" button next to one of your journals    
BU.1.2.1   a new browser tab opens, titled "Journal update request form for ..."
BU.1.3 Attempt to edit the Journal title, Alternative title, ISSN (print), ISSN (online)    
BU.1.3.1   You cannot
BU.1.4 Make some changes to the editable questions (make a note of what those changes are)    
BU.1.5 Click on 'Review your answers' in the section stepper on the right and submit the form    
BU.1.5.1   The form is submitted successfully and you are taken to the 'My update requests' tab
BU.1.5.2   A flash message tells you that your update has been submitted. Another flash message says that a confirmation email has been sent.
BU.1.6 clear the flash messages    
BU.1.7 Click on the "My journals" tab    
BU.1.7.1   The journal you submitted an update for now says "View update" by it
BU.1.8 Go back to the "My Update Requests" tab    
BU.1.8.1   The journal for which you submitted an update is listed here
BU.1.9 Click "Edit" on the update request you created above    
BU.1.9.1   You can see the edit form for the update request
BU.1.9.2   Your changes implemented above are present
BU.1.10 Make another change to the form and submit    
BU.1.10.1   Your changes are submitted successfully.
BU.1.11 Click "Delete"    
BU.1.11.1   A pop-up appears asking you if you are sure you want to delete the update request.
BU.1.12 click 'No'    
BU.1.13 Click 'Delete' and 'Yes, delete it'    
BU.1.13.1   You are shown a message 'Your application has been deleted'
BU.1.14 Click the link to take you back to the dashboard and click on 'My update requests'    
BU.1.14.1   The update request no longer appears on this tab
BU.1.15 create a new update request for one of your journals and submit it    
BU.1.16 Open a new session in a different browser or a private (incognito) window    
BU.1.17 Log in as an administrator and go to the dashboard    
BU.1.18 Go to the Update Requests Search page    
BU.1.18.1   The update request you submitted above is visible
BU.1.19 Click "Review Update"    
BU.1.19.1   A new browser tab opens with the update request form
BU.1.19.2   The top of the form contains a table showing the difference between the journal and the update request
BU.1.19.3   The changes in the difference table match the changes you made when submitting the update request
BU.1.20 Click on one of the links in the left column of the difference table    
BU.1.20.1   You are scrolled down to the corresponding change and the change is highlighted in black under the question
BU.1.21 Go back to the top of the form and change the status to "In Progress" and save. Do not make any other changes at this point    
BU.1.22 Go back to the other browser window, where you are signed in as a publisher    
BU.1.23 Go to the Publisher's Update Requests tab    
BU.1.24 Go back to the browser window where you are logged in as admin    
BU.1.25 Edit the update request, and change the status to "Revisions Required"    
BU.1.25.1   flash notification confirms that the publisher has been alerted
BU.1.26 Go back to the other browser window, where you are signed in as a publisher    
BU.1.27 Go to the 'My update requests' tab. You may need to reload the page.    
BU.1.27.1   Your update request is editable again and the black button now says 'Edit'
BU.1.28 Make an edit to the update request    
BU.1.29 Go back to the browser window where you are logged in as admin    
BU.1.30 Edit the update request, and Reject it    
BU.1.31 Go back to the My update requests tab in the other browser window, where you are signed in as a publisher    
BU.1.31.1   The status of the update requests has changed to 'Application rejected'.
