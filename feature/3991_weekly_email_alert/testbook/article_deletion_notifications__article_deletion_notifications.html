1. Article Deletion Notification Generation

Test context
  • role: admin
Setup:

As an admin or through the API, delete one or more articles belonging to a publisher.

Ensure the articles were deleted within the last 7 days.

ID Action Expected Result  
L.1.1 Log in as the publisher who owned the deleted articles after an hour (scheduler configured to run hourly for testing).    
L.1.2 Check the notifications (bell icon or /notifications).    
L.1.2.1   A new notification is present with the title "Deleted articles in your journal(s) this week".
L.1.2.2   The notification content lists the titles and details of the deleted articles.
L.1.3 Verify that if multiple articles were deleted for the same publisher, they are grouped into a single notification.    

2. No Notifications when no articles deleted

Test context
  • role: admin
Setup:

Ensure no articles have been deleted in the last 7 days (or for a specific set of publishers).

ID Action Expected Result  
L.2.1 Check notifications for publishers.    
L.2.1.1   No new deletion notifications are received.
