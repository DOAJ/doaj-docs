1. Preparation

Test context
  • role: Any Authenticated User
ID Action Expected Result  
R.1.1 Activate the testdrive setup for this test suite, by using the following url

Application Link: http://testdoaj.cottagelabs.com//testdrive/autocheck 		   
R.1.1.1   You receive the testdrive setup details, including an admin account for you to use, and an application and journal which have been autochecked for testing
R.1.2 Log into DOAJ using the admin credentials supplied by the testdrive. You may want to do this in a private browsing window, so you can remain logged in to your own account in your main window.

Application Link: http://testdoaj.cottagelabs.com//account/login 		   
R.1.2.1   You are logged in as an admin user
R.1.3 You may now proceed to the tests below    

2. Application autochecks

Depends On
  • Autocheck : Autocheck : Preparation
Test context
  • role: Admin
ID Action Expected Result  
R.2.1 Go to the Application admin_url supplied by the testdrive setup    
R.2.1.1   You are looking at a test Application created for this test
R.2.2 Scroll down to find the ISSN (print) and ISSN (online) fields    
R.2.2.1   The Print ISSN field is annotated with a green tick, and text which says that ISSN is fully registered at ISSN.org
R.2.2.2   The Electronic ISSN field is annotated with an orange cross, which says the ISSN is not registered at ISSN.org
R.2.3 Click on one of the "see record" links in the annotation    
R.2.3.1   The link is opened in a new window/tab
R.2.3.2   You are taken to the ISSN.org record. Note that for the purposes of this test, this is a random record on ISSN.org, unrelated to the actual record in DOAJ.
R.2.4 Close the ISSN.org window/tab and return to the application form    
R.2.5 Scroll to the "Best Practice" section of the application form, and look at the "Long-term preservation services" question    
R.2.5.1   5 checks are visible attached to this question, for: CLOCKSS, LOCKSS, Internet Archive, PMC and PKP PN
R.2.5.2   CLOCKSS is annotated with a red exclamation, saying it is archived but not selected in the form
R.2.5.3   LOCKSS is annotated with a green tick, saying it is currently archived
R.2.5.4   Internet Archiive is annotated with a red cross, saying it is archived but not current
R.2.5.5   PKP PN is annotated with a red cross, saying it is not archived
R.2.5.6   PMC is annotated with a grey info symbol, saying it is not currently recorded by Keepers
R.2.6 Click on one of the "see record" links in the annotation    
R.2.6.1   The link is opened in a new window/tab
R.2.6.2   You are taken to the ISSN.org record. Note that for the purposes of this test, this is a random record on ISSN.org, unrelated to the actual record in DOAJ.
R.2.7 Close the ISSN.org window/tab and return to the application form    
R.2.8 Scroll to the top of the application form    
R.2.8.1   There is text which tells you when the Autochecks were made, and an option to Hide All Autochecks
R.2.9 Click "Hide All Autochecks", then scroll through the form    
R.2.9.1   The autochecks are all hidden
R.2.10 Return to the top of the application form and click "Show All Autochecks", then scroll through the form    
R.2.10.1   The autochecks are all visible again
