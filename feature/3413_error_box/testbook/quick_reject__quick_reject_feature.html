Quick Reject: Quick Reject Feature

Download Test Suite CSV

1. Edit a Rejected Application

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select a "Rejected" application (e.g. by selecting "Rejected" from the "Application Status" facet on the left), and click "View finished application"
  • The applicaiton review page is shown
  • The application is in status "Rejected"
  • There is no "Quick Reject" button shown on the page
Click the "Unlock and Close" button

2. Edit an Accepted Application

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select an "Accepted" application (e.g. by selecting "Accepted" from the "Application Status" facet on the left), and click "View finished application"
  • The applicaiton review page is shown
  • The application is in status "Accepted"
  • There is no "Quick Reject" button shown on the page
Click the "Unlock and Close" button

3. Select a pre-determined reject reason only

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select an application that is not either "Accepted" or "Rejected" (e.g. by selecting "Pending" from the "Application Status" facet on the left), and click Review application
  • The applicaiton review page is shown
  • The "Quick Reject" feature is shown above the application form
Select a reason for rejection from the pull-down menu. Select any reason except "Other". Do not enter any text in the additional information box
click "Quick Reject"
  • A flash message containing the text "This application was rejected with the reason 'X'" is displayed, where X is the reason you selected for rejection.
  • A flash message containing the text "Sent email to 'X' to tell them that their journal application was rejected" is displayed, where X is the journal contact
  • The applicaiton review page is shown
  • The application is in status "Rejected"
  • There is no "Quick Reject" button shown on the page
  • There is a note attached to the application at the current date/time which contains the text "[User name:] This application was rejected with the reason 'X'" is displayed, where X is the reason you selected for rejection.
Click the "Unlock and Close" button

4. Select a pre-determined reject reason and add custom text

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select an application that is not either "Accepted" or "Rejected" (e.g. by selecting "Pending" from the "Application Status" facet on the left), and click Review application

  • The "Quick Reject" feature is shown above the application form

Enter some free-text in the additional information box
click "Quick Reject"
  • A flash message containing the text "This application was rejected with the reason 'X - Y'" is displayed, where X is the reason you selected for rejection, and Y is the free text you entered
  • A flash message containing the text "Sent email to 'X' to tell them that their journal application was rejected" is displayed, where X is the journal contact
  • The applicaiton review page is shown
  • The application is in status "Rejected"
  • There is no "Quick Reject" button shown on the page
  • There is a note attached to the application at the current date/time which contains the text "This application was rejected with the reason 'X - Y'" is displayed, where X is the reason you selected for rejection, and Y is the free text you entered
Click the "Unlock and Close" button

Action Expected Result
