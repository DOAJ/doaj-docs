1. Create a new user

Test context
  • role: admin
Action Expected Result
Go to the user search in admin at /account
Click "create new user" in the top left
  • You are taken to the new user form
  • The "api" role and "publisher" role are pre-filled in the Roles field.
  • The ID box is prefilled with an alphanumeric value
Click in the "Roles" field
  • A pull-down menu appears with all the allowed roles
Click in the 'ID' field
  • It is editable
  • Change the ID to something memorable
Enter test values into each field in the form
Click "Create User"
  • You are taken back to the /account
  • You see a message Debug mode - url for verify is /account/reset/f1b6b123b5f34cd0aa81807a90a096f2
  • /account/reset/f1b6b123b5f34cd0aa81807a90a096f2 is a link and is clickable DO NOT CLICK IT YET
  • You see a message ' Account created for dom+test5@doaj.org. View Account: /account/testingID'
  • /account/testingID is a link and is clickable DO NOT CLICK IT YET
  • The ID (/testerID in the example above) matches the ID you chose on line 11
Click on the link that starts: Debug mode - url for verify is...
  • You are taken to the page to set a password
  • You see the name of your user at the top
Enter different values into each password field
Click "Update"
  • You see an error message requiring that the passwords match
Enter matching passwords into the password fields
Click "Update"
  • You are redirected to the home page
  • There is a flash message that says: New password has been set and you're now logged in.
Log out
Log back in with your Admin account
Go back to /account/
Enter the *name* of your test user into the search box
  • Your new user is listed as the only result in the search
  • The user has all the properties as you specified them in the form
  • The user's account created and last modified dates are correct
Clear your search
Enter the *ID* of your test user into the search box
  • Your new user is listed as the only result in the search
  • The user has all the properties as you specified them in the form
  • The user's account created and last modified dates are correct

2. Edit your own user account (publisher)

Test context
  • role: publisher
Action Expected Result
Log in to your publisher account
Go to Settings under My Account
  • Your account page is shown
  • Your ID and email address are displayed
  • You can see your user roles but you cannot edit them

3. Edit your own user account (admin)

Test context
  • role: admin
Action Expected Result
Log in to your admin account
Go to Settings under My Account
  • Your account page is shown
  • Your ID and email address are displayed
Log back in to your admin account and go to Settings
Check the "yes, I'm sure" checkbox
Click "Delete"
  • A flash message tells you your user account has been deleted
  • You are logged out of DOAJ
Recreate your user account either from another administrator account, the command line, or data restore

4. Edit another user's account

Test context
  • role: admin
Action Expected Result
Select a user account to modify
Locate that user in the user search, and select "Edit this user"
  • The user's account page is displayed
  • A warning alerts you 'NOTE you are editing a user account that is not your own. Be careful!'
Check the "yes, I'm sure" checkbox
Click "Delete"
  • A flash message tells you the user account + number has been deleted
  • You are redirected to the account search page
Put the username of the account you deleted into the search box
  • There are no results to display
