1. Accept a New Application

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select a new application that can be accepted (e.g. is in a status such as Ready) and click "Review application"
Complete any required fields, and then change the Application Status to "Accepted"
Click "SAVE"
  • A flash message indicates that a new journal was created, and links to that journal
  • A flash message indicates that the application has been updated-
  • A flash message indicates that the journal has been associated with a new or existing user account
  • A flash message indicates an email has been sent to the account holder to tell them journal was accepted.
  • The Application Status field says "Accepted", and a note at the top of the form says "Please note you cannot edit this application as it has been accepted into the DOAJ."
Click the "Unlock and Close" button
Go to the mailtrap.io inbox for the test environment you are working in
  • There should be a suitable email to the journal owner indicating that their journal has been accepted
  • There should be a suitable email to the journal contact indicating that their journal has been accepted

2. Accept an Update Request

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
Select an update request that can be accepted (e.g. is in a status such as Ready) and click "Review update"
Complete any required fields, and then change the Application Status to "Accepted"
Click "SAVE"
  • A flash message indicates that an existing journal was updated and links to that journal
  • A flash message indicates that the journal has been associated with an existing user account
  • A flash message indicates an email has been sent to the owner of the journal
  • The Application Status field says "Accepted", and a note at the top of the form says "Please note you cannot edit this application as it has been accepted into the DOAJ."
Click the "Unlock and Close" button
Go to the mailtrap.io inbox for the test environment you are working in
  • There should be a suitable email to the journal owner indicating that their update request has been accepted

3. Reject a New Application

Test context
  • role: admin
Action Expected Result
Go to the "Applications/Update Requests" page in the Administration area
