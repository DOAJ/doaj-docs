Article XML Upload: Crossref 4.4.2

Download Testset

1. Upload an XML file which does not meet the Crossref schema

Test context
  • role: publisher
ID Action Expected Result  
K.1.1 Go to the "Upload Article XML" tab in the "Publisher Area"    
K.1.2 Select "Choose file" and select a file from your computer which is XML but not of the Crossref schema. You may use the file "schema_invalid.xml" from the Crossref folder in resources for this test.

Test Resource: https://raw.githubusercontent.com/DOAJ/doaj/hotfix/single_source_isolang/doajtest/xml_upload_test_package/Crossref/schema_invalid.xml 		   
K.1.3 Choose "Crossref 4.4.2 XML" from "Format of the file" dropdown    
K.1.4 Click "Upload"    
K.1.4.1   A flash message appears at the top of the screen indicating an error has occurred
K.1.4.2   Your file is shown in the "History of uploads" with status "processing failed" and a suitable entry in the "Notes" (you may need to reload the page). Check that the explanation link goes to a suitable reason and resolution for the problem.

4. Upload a file containing ISSNs the publisher does not own

Test context
  • role: publisher
ID Action Expected Result  
K.4.1 Ensure that the ISSNs 0000-0002 and 0000-000X are in DOAJ attached to one or more journals (e.g. one with 0000-000X AND 0000-0002 or two, one with 0000-0002 and another with 0000-000X). None of the journals with these ISSNs should be owned by the publisher    
K.4.2 Go to the "Upload Article XML" tab in the "Publisher Area"    
K.4.3 Choose "Crossref 4.4.2 XML" from "Format of the file" dropdown

K.4.5.1   A flash message appears at the top of the screen indicating a successful upload
K.4.5.2   Your file is shown in the "History of uploads" with status "pending"
K.4.6 wait a short amount of time for the job to process, then reload the page (do not re-submit the form data). If the job remains in "pending", reload the page until the status changes.    
K.4.6.1   Your file is shown in the "History of uploads" with status "processing failed" and a suitable entry in the "Notes". Check that the explanation link goes to a suitable reason and resolution for the problem.
K.4.7 click on "(show error details)" for the record in the "History of uploads"
